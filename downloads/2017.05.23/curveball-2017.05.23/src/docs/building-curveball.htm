Introduction

This document describes how to build the version of Raytheon BBN Technologies' BBN Curveball specified in the title of this document.

This document is intended for developers who want to extend or modify BBN Curveball in some way. (Ordinary users will usually find it more convenient to install a pre-built version of BBN Curveball than to build their own.)

Note that installing the tools needed to build BBN Curveball may require system privileges on many operating systems.

This document only describes how to build BBN Curveball for Ubuntu 12.04 on the amd64 platform. BBN Curveball also supports Windows 7, Windows 8, Android, and MacOSX, and future releases will contain detailed instructions for building for those platforms.

Getting the BBN Curveball source

The source for BBN Curveball is available from curveball.nct.bbn.com.

Make sure that the version you download matches this document. If you choose to download and use a different version of the source, then you can follow the directions for getting and unpacking the source in this document, and then continue with the appropriate instructions that were bundled with the version you downloaded.

The BBN Curveball source is released as a gzip'd tar file. The name of the file will have the form curveball-VERSION.tgz, where VERSION is a string of the form YYYY.MM.DD that represents the date of the release. In the rest of this document, we will use the string VERSION to represent the date of the release.

In order to open it, you need to have a version of tar that is able to decompress the gzip format, or you will need to have a version of gzip. These tools are included by default on Linux and MacOSX, but on Windows it may be necessary to download a third-party tool. We recommend mingw, for building on Windows 7.

Linux - Ubuntu 12.04 (amd64)

Curveball has been primarily developed and tested on Ubuntu 12.04 on the amd64 architecture. It has also been built and tested successfully on the i386 architecture. It has not been tested on architectures with different endianess.

NOTE: for this version of BBN Curveball, it is generally a good idea to build on a virtual machine or dedicated host because the build procedure may download and install or update any packages that are required by the build process or by BBN Curveball. Also note that the build procedure may also install BBN Curveball software on the build machine, and may alter configuration files. Building BBN Curveball may modify the configuration of the build machine, even if BBN Curveball is not installed. This will be fixed in a future release.

After downloading the release tarball with the desired VERSION, and placing it in working direction: 

    tar zxf curveball-VERSION.tgz
    cd curveball-VERSION/src
    make debfile

This will create a Debian package file for the current architecture, with a name with the prefix curveball-VERSION, in the directory curveball-VERSION/src.

Note: during the build of the BBN Curveball click elements, you may see messages like the following: 

    ...
    CXX decodertest.cc
    /bin/bash: decodertest.d: No such file or directory
    CXX dr2dpdecoder.cc
    /bin/bash: dr2dpdecoder.d: No such file or directory
    CXX dr2dpdecodertest.cc
    /bin/bash: dr2dpdecodertest.d: No such file or directory
    ...

These messages are harmless and do not indicate that the build has failed.

Notes

There are no prerequisites that need to be installed or configured before building BBN Curveball on Ubuntu 12.04. The build procedure will download and install any packages that are required by the build process and are not already installed. Note that this requires network access and sudo.

In addition to creating the Debian package, the Curveball scripts, executables, and other files are copied to curveball-VERSION/build.
